Reúne dados dispersos numa base confiável e consultável.

O Aqueduto importa os seus dados, conduz por etapas conectadas (como a água num aqueduto), mede a qualidade e organiza as métricas — para o Power BI e para agentes de IA. Cada etapa alimenta a seguinte, e tudo é calculado sobre os dados que você importa.

Transparência: este protótipo não usa números fixos nem inventados. Todos os resultados são calculados a partir do arquivo importado (ou de um conjunto de exemplo rotulado). O agente só responde com base nesses dados.

Roda com os seus dados

conector integrado (Databricks, ADF, Fabric), API REST ou CSV — o software calcula sobre o que entra.

Confiável e medido

mede nulos, duplicatas, outliers e a confiabilidade sobre o dado real.

Uma definição por métrica

o modelo semântico dá uma definição única de cada métrica para toda a empresa.

1 · Conectar dados

Entrada: fontes da empresa  ·  Saída: camada Bronze → alimenta a etapa 2.
O que faz: traz os dados para dentro do hub. Três formas — todas orientadas à stack da vaga: conector integrado (Azure Databricks, ADF, Microsoft Fabric, ERP/CRM), API REST, ou arquivo CSV.
Clique num conector para importar (dados de exemplo, rotulados). Em produção, é a conexão real com a plataforma.

2 · Camadas de dados

Entrada: dado bruto (etapa 1)  ·  Saída: tabelas Gold → alimentam Qualidade e Dashboard.
O que faz: organiza o dado em camadas com propósito. Bronze = dado cru como veio (histórico auditável). Silver = limpo, sem duplicatas e validado por data contracts. Gold = tabelas de negócio prontas. Contado sobre o seu dado.

6 · Agentes & IA operacional

Entrada: métricas do Gold (etapa 4)  ·  Saída: respostas e ações baseadas no dado.
O que faz: a empresa pluga a própria chave de IA. O agente responde apenas com base no dado importado — se não estiver nos dados, ele diz que não consta. Sem inventar.
